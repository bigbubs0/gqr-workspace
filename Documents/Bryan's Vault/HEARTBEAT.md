# HEARTBEAT.md - Periodic Task Checklist

## Every Session

- [ ] Check `/inbox/` for new files to process (CVs, JDs, voice notes, articles)
- [ ] Check TODO.md for URGENT items
- [ ] Process any pending candidate submissions or feedback
- [ ] Write session summary to `/memory/YYYY-MM-DD.md` with date

## Weekly (Mondays)

- [ ] Summarize active job orders by status across all accounts
- [ ] Flag candidates in pipeline with no activity >7 days
- [ ] Scan Notion Funding DBs for new entries (flag Series B+ at ICP companies)
- [ ] Scan Notion Layoffs DB for new entries (flag talent availability for active searches)
- [ ] Check for positive clinical data readouts in tracked companies
- [ ] Report any Phase 2 to Phase 3 transitions in tracked companies
- [ ] Check for new job postings at target companies (CMO, VP Clinical, Head PV, Director Biometrics)
- [ ] Review archive trigger conditions on all active company files
- [ ] Update Company Core if new intelligence surfaces
- [ ] Run weekly green/red sync audit using `/config/sync-manifest.json` and `/review/weekly-sync-audit-template.md`
- [ ] Run weekly green/red signal audit using `/config/signal-health-manifest.json` and `/review/weekly-signal-automation-audit-template.md`

## Monthly (First Monday)

- [ ] Audit active vs dead accounts - update USER.md account status
- [ ] Review and update Priority Signals list if market conditions shifted
- [ ] Review skills in `/skills/` - flag any that need updating based on workflow changes
- [ ] Check MEMORY.md for patterns worth surfacing (recurring candidate types, common objections, market shifts)
- [ ] Verify `/config/notion-databases.json` IDs are still accurate

## Quarterly

- [ ] Full account portfolio review - owned, team, prospects, dead
- [ ] Audit recruiting frameworks usage - are INTEL/SPECIFICS/SCREEN/RATESSS/CLABVISSS being applied consistently?
- [ ] Review content strategy - RecruitRx topics, LinkedIn performance
- [ ] Clean `/archive/` - remove anything >12 months old with no reactivation potential
